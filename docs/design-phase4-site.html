1. ゴール像と導線

ナルエビ方式の導線。X が集客し、サイトが本文を受け持つ。

図1: X ポスト → サイトの導線（ナルエビ方式）。実線=4a、破線=4b 以降。

  • X が集客、サイトが本文。ポスト単体でも意味が通る文面にする（リンク踏まなくても価値がある＝フォロー維持）
  • ナルエビちゃんニュースから借りる型: トピックカード最新順フィード / キャラ視点セクション / 個別ページ＋OGP / RSS。借りない型: 難易度4段階は3段階に簡素化、既読ボタン等は後回し（4c）

2. アーキテクチャ（最速案）

現状: .github/workflows/generate-news.yml は workflow_dispatch のみ、成果物は CI artifact 止まり（コミットもデプロイもなし）。サイト層はゼロから。

推奨: 1本のワークフローに直列で追加（新サービス契約ゼロ、リポジトリ完結）

図2: 1本の GitHub Actions ワークフローに直列で追加する5ステップ。

  1. npm run start … 既存パイプライン（deepseek）
  2. データ永続化 … output/ の md・topic_candidates・selection_trace・fact_ledger ＋ 新規 articles_{date}.json を data/{date}/ にコミット
  3. npm run build:site … 新規 src/site/build.ts（zero-dep TS）が data/ 全日分 → dist/site/ に静的HTML生成
  4. Pages デプロイ … actions/upload-pages-artifact + actions/deploy-pages
  5. npm run post:x … 新規 src/site/postToX.ts が日次ダイジェストを1本投稿

設計上の要点

  • articles_{date}.json（新規・小改修）: renderMarkdown.ts が受け取るのと同じ構造化データ（SummarizedArticle[]）を JSON でも書き出す。サイトビルダーが Markdown をパースし直すのを避け、スキーマ正本を src/types.ts に一本化するため。パイプライン側の唯一の改修点
  • data/ ディレクトリ: 現在 output/*.md,*.json は .gitignore 対象。日次成果物は data/YYYY-MM-DD/ にコミットして履歴＝アーカイブにする（サイトの全日分ビルドの源泉）
  • zero-dep 原則の維持: サイトビルダーも X 投稿も Node 標準＋TS のみ。X の OAuth 1.0a 署名は node:crypto の HMAC-SHA1 で自前実装可能（外部SDK不要）
  • X 投稿は最後: ビルド/デプロイ失敗時に投稿だけ先行する事故を防ぐ。投稿失敗はワークフロー失敗にする（無言の取りこぼし防止）
  • Secrets 追加: X_API_KEY / X_API_SECRET / X_ACCESS_TOKEN / X_ACCESS_SECRET

3. X bot 仕様

課金前提（要一次確認・重要）

X API は 2026年2月に無料枠廃止・従量課金化（第三者情報）。設計はこの前提:

  • テキスト/メディアのみのポスト: $0.015/本
  • URLを含むポスト: $0.20/本 ← リンク戦略がコストを支配する
  • 読み取りは使わない（投稿のみ）

文面ルール

  • 記事の見出し・リードから機械的に組み立てる（新規生成しない＝捏造リスクゼロ）
  • ハッシュタグは付けない（bot臭・コスト対効果なし）。絵文字はキャラ絵文字1つまで
  • 投稿時刻は生成完了直後（毎朝10時台 JST）

4. 情報設計（サイトマップ）

図3: サイトマップ。朱枠=X ポストの主要リンク先、破線=4b で追加。

  • フィード並び順: 日付降順 → 日内は publish_priority（high→low）→ newsworthiness_score。renderMarkdown.ts と同一ロジック
  • パーマリンクに連番 n を使う理由: topic_key は中国語でURLに不向き、ローマ字化は依存が増える。日付＋掲載順は決定的で短く、X の字数も食わない（トレードオフ: 過去日の再生成で順序が変わると URL が壊れるため、data/ にコミットした articles JSON を正とし再生成しない運用で担保）
  • カテゴリ絞込は 4c（クライアントJS）。4a はタグ表示のみ

守るラインサイトでの担保
出典必須全カードにソース行。空ならビルド失敗（build.ts のバリデーション）

官庁比率 ≦ 50%サイトは表示のみだが、/about に編集方針として明記し読者にも開示
