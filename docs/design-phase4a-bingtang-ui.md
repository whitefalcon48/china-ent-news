## 2. デザイントークン

最終キャラ画像下部のパレット実測値（Fable がピクセル実測: 氷青 #A7CDDF / 紅 #C12B23 / アンバー #CD7019 / 濃紺 #1F3043 / アイボリー #F0E6DA）を基準に展開する。

```css
:root{
  /* パレット原色（キャラ画像実測） */
  --bt-ice:      #A7CDDF; /* 氷青（髪・キー色） */
  --bt-red:      #C12B23; /* 紅（山査子。主アクセント） */
  --bt-amber:    #CD7019; /* アンバー（瞳。確度A・SNS） */
  --bt-navy:     #1F3043; /* 濃紺（ロゴ・見出し） */
  --bt-ivory:    #F0E6DA; /* アイボリー（カテゴリチップ地） */
  /* 展開色 */
  --bt-ice-50:   #F6FAFC; /* ページ地 */
  --bt-ice-100:  #EAF4FA; /* 吹き出し地・薄地 */
  --bt-ice-200:  #DEEEF6; /* ヘッダ地 */
  --bt-ice-600:  #4E8FAE; /* データpip・補助リンク */
  --bt-red-dark: #A32017; /* リンクhover */
  --bt-amber-50: #FBF1E2; /* 警告バッジ地 */
  --bt-amber-900:#7A4A10; /* 警告文字・確度A文字 */
  --bt-text:     #2A3948; /* 本文 */
  --bt-muted:    #6E7E8C; /* 弱文字 */
  --bt-border:   #DCE8EF; /* 罫線 */
  --bt-card:     #FFFFFF;
  --bt-silver:   #9AA7B1; /* 確度B縁 */
  --bt-gray:     #C2CBD2; /* 確度C縁 */
}
```

- 文字色は本文 `--bt-text`、見出し・カードタイトル `--bt-navy`、リンク `--bt-red`（下線なし、hover 下線＋ `--bt-red-dark`）
- アンバーは小さい本文文字に使わない（白地でのコントラスト不足。チップ縁・警告地の文字は `--bt-amber-900` を使う）
- フォント: `"Hiragino Maru Gothic ProN","Hiragino Kaku Gothic ProN","Yu Gothic UI","Yu Gothic","Meiryo",sans-serif`（既存モック踏襲）
- 文字サイズ: 本文 16px 基準。カードタイトル 1.08rem/700、個別ページ h1 1.35rem/800、リード .92rem、メタ・チップ .72〜.78rem。行間: 本文 1.85、タイトル 1.6
- 角丸: カード 14px、チップ 999px、吹き出し 4px 14px 14px 14px（左上のみ角）、ボタン 999px
- 影: `0 1px 3px rgba(31,48,67,.08)`（カード）。ヘッダ・フッタは影なし
- 余白: 4px グリッド。カード内 padding 18px 20px、カード間 gap 20px、セクション間 40px
- ライトテーマのみ（確定事項）

## 4. 写真なし記事カード（フィード用・本設計の中心）

写真の視覚アンカーを次の3点で代替する:

1. **種別色トップバー**: カード上辺に高さ4pxのバー（OFFICIAL=navy / NEWS=red / DATA=ice-600）。一覧を色でスキャンできる
2. **チップ列**: 種別・カテゴリ・確度・鮮度を1行目に固定順で並べる
3. **アバター吹き出し**: ビンタンの標準表情の丸アバター36px＋氷青地の吹き出し。カードごとに繰り返されてリズムを作る

### カード構造（上から順）

```
[種別色トップバー 4px]
チップ列: 種別 / カテゴリ / 確度 / 鮮度 …右端に日付(YYYY/M, 等幅)
タイトル（h2→個別ページへのリンク。navy、折返し許容・切詰め禁止）
リード（.92rem。CSSで3行clamp。全文は個別ページ）
ソース構成: pips 4色＋件数 ＋（公式のみの場合）警告バッジ
[ビンタンの注目ポイント] アバター36px＋見出し付き吹き出し（氷青地）。PC・スマホとも必須表示。CSSで4行clamp
ソース行: 「ソース: 媒体名リンク、…」（.78rem。全媒体を列挙、省略しない）
フッタ行: 「しっかり読む →」（red・右寄せ・個別ページへ）
```

- `editor_comment`（旧「ビンタンからのひとこと」）、「何が起きた？」「反応・見られ方」「日本語圏では見えにくいポイント」はフィードカードに**出さない**（個別ページのみ）
- 一覧カードのアバターだけを置いて見出し・本文を省略する表示は禁止。PCでも必ず「ビンタンの注目ポイント」の見出しと `why_it_matters` 本文を同じ吹き出し内に表示する
- 一覧カード内のアバターは「ビンタンの注目ポイント」枠の1つだけとし、タイトル横・チップ列などに同じアバターを重複表示しない（ユーザー確認 2026-07-17）
- clamp は CSS の表示上の省略（`-webkit-line-clamp`）であり、テキスト自体は改変しない
- タイトルは何行でも全文表示（信頼性・可読性優先）
- Xシェアリンクは個別ページのみ（フィードカードは「しっかり読む」1導線に絞る）

## 6. ページ仕様

### 共通ヘッダ（トップ・日次・アーカイブ・About）

- 地: `--bt-ice-200`（下辺 `--bt-border` 1px）
- 左: ロゴ「冰糖<span style="color:red系">日报</span>」（冰糖=navy・日报=red、1.6rem/900）／直下に「ビンタンちゃんデイリー」（red .8rem）／タグライン「中国エンタメの現地温度を、日本語で。」（muted .85rem。「現地温度」のみ red）
- 右: キャラ透過バスト＋吹き出し「今日のわたしが気になる中国エンタメ情報です！」（白地・角丸・.85rem）
- 日付バッジ: 「2026年7月13日（月）」形式、ロゴ列の下（トップ・日次のみ。データの日付を表示。当日と偽らない）
- ナビ: 「最新」「アーカイブ」「このサイトについて」の3リンク（白地バー・現在地は red 下線2px）。**カテゴリタブは4cで追加**

### 個別ページ用の小ロゴバー

- 高さ52px・`--bt-ice-200` 地。左にロゴ小（1.1rem）→トップへ、右に「← 2026-07-13 の一覧へ」リンク

### トップ `/`

ヘッダ → フィード（直近10トピック。日付見出し「7月13日（月）」で区切る）→ 「過去の記事はアーカイブへ →」導線 → 凡例 → フッタバナー → サイトフッタ

### 日次 `/archive/{date}/`

ヘッダ（日付バッジ=当該日）→ その日の全カード（フィードと同じコンパクトカード）→ 凡例 → フッタバナー → サイトフッタ

### 個別 `/t/{date}/{n}/`

小ロゴバー → 記事（1カラム・max-width 720px）:

```
チップ列（種別/カテゴリ/確度/鮮度/日付）
h1 タイトル
リード（全文）
ソース構成 pips ＋（該当時）警告バッジ
「何が起きた？」見出し＋全文
[ビンタンの注目ポイント] 吹き出し（全文）
「反応・見られ方」（データに存在する時のみ）
「日本語圏では見えにくいポイント」（同上）
ソース行（全媒体リンク）
Xシェアリンク（intent/tweet）
前後ナビ: 「← 前の記事」「次の記事 →」（同日内）＋「この日の一覧へ」
```

個別ページのキャラコメントは、見た目上は1つの「ビンタンの注目ポイント」枠に統合する。

```
[丸アバター36px] ビンタンの注目ポイント
why_it_matters（全文）

──────── 薄い区切り線（editor_comment がある場合のみ）

editor_comment（短い締めコメント。独立見出し・2つ目のアバターなし）
```

- `why_it_matters` と `editor_comment` のJSON項目は互換性維持のため変更しない。サイトのHTML表現だけを統合する
- `editor_comment` が空なら区切り線ごと表示しない
- `why_it_matters` が空で `editor_comment` だけ存在する場合も、同じ「ビンタンの注目ポイント」枠で表示する
- 「ビンタンからのひとこと」という独立見出し・独立カードはサイトでは使用しない

セクションの掲載順は `renderMarkdown.ts` の出力順と同一にする（本文の順序も改変しない）。

### アーカイブ `/archive/`

ヘッダ → 日付リスト（新しい順。「2026-07-13（10本）」形式で日次ページへ）→ サイトフッタ

### About `/about/`

ヘッダ → ①ビンタン紹介（`design-phase4-character.md` §4.6 の「About用（導入段落）」を一字どおり使用。丸アバター64px添え）
→ ②このサイトについて（編集方針の要約: 現地温度とズレを埋める目的・翻訳サイトではない旨。`editorial-character.md` の「サイトの核」に準拠）
→ ③出典と確度の読み方（チップ凡例＋「公式発表のみ」警告の意味）
→ ④運営情報（AI生成である旨・Falさん（人間）監修・お問い合わせ先は将来追加）→ サイトフッタ

### フッタバナー（トップ・日次のみ）

`--bt-ice-100` 地・角丸14px。アバター48px＋「気になるニュースは『しっかり読む』から全文をどうぞ。過去の記事はアーカイブにありますよ！」＋「このサイトについて →」ボタン（red地・白字）

### サイトフッタ（全ページ）

navy 地・白字 .78rem。「冰糖日报（ビンタンちゃんデイリー）／記事はAIが収集・生成し、人間が監修しています／© 2026 冰糖日报」＋ About・アーカイブへのリンク。RSS リンクは 4b で追加

## 8. 状態別仕様

| 状態 | 仕様 |
|---|---|
| 通常 | §4 のとおり |
| 公式のみ（公式n≥1・媒体0・SNS0・データ0） | 警告バッジをソース構成の行内に必ず表示。省略・縮小・低彩度化しない。カード構造は通常と同一 |
| 複数ソース | pips が複数点灯。ソース行に全媒体をリンク列挙（数を偽らない・省略しない） |
| 長いタイトル | 折返しで全文表示（フィードでも切詰めない） |
| 長いリード／注目ポイント | フィードのみ CSS clamp（リード3行・注目ポイント4行）。個別ページは全文 |
| `editor_comment` あり | 個別ページの統合コメント枠内で、薄い区切り線の後に表示。別見出し・別アバターは作らない |
| `editor_comment` なし | 区切り線を含め、締めコメント領域を表示しない |
| 記事0件の日 | 日次ページ: カードの代わりに空状態ボックス「この日は記事をお届けできませんでした。収集または生成に失敗したためです。前日までの記事はアーカイブからどうぞ。」（装飾最小・キャラ装飾なし）。トップ: 直近の生成成功日までを表示（日付見出しがあるため偽りにならない） |
| 一部取得失敗 | 生成できた記事だけを表示。件数の水増し・ダミーカードは作らない |
| 画像読込失敗 | §7 の絵文字代替 |
| 「反応・見られ方」等がない記事 | セクションごと非表示（見出しだけ残さない） |

## 12. 検証手順と受け入れ基準

### 検証手順

1. `npm run check`
2. `data/2026-07-13/` に実データ（articles JSON）を配置して `npm run build:site`
3. ローカルHTTPサーバ（例: `npx serve dist/site`）で全ページを開く
4. PC幅 1280px とスマホ幅 375px でトップ・日次・個別のスクリーンショットを取得し、ユーザーに提示
5. 異常系: ①ソース行空データ→ビルド失敗 ②素材PNG削除→絵文字fallback ③記事0件の日→空状態文言 ④公式のみ記事→警告バッジ

### 受け入れ基準

**機能**
- 5種のページが生成され、全内部リンク（カード→個別、前後ナビ、ナビ3リンク、日付見出し→日次）が切れていない
- ソース行空カードでビルドが失敗する
- OGP メタ（title/description/既定画像）が全ページに入る

**編集ガードレール**
- 全カード・全個別ページにソース行と ソース構成表示がある
- 公式のみ記事の警告バッジが弱められていない
- 本文・コメントのテキストが articles JSON と一致（改変・追加・省略なし。CSS clamp のみ許容）
- サイト上の固定文言（吹き出し・空状態・フッタ等）が本書記載の確定文面と一字一致し、口調規定（です・ます＋ね/よ）に適合
- データにない要素（写真・ダミー画像・相対時刻・存在しないセクション）が出ていない
- 一覧の全カードに「ビンタンの注目ポイント」見出しと `why_it_matters` が表示される（空データ時を除く）
- 個別ページの `why_it_matters` と `editor_comment` は1つのコメント枠に入り、独立した「ビンタンからのひとこと」枠がない

**視覚品質**
- 375px〜1280px で横スクロールなし
- 本文文字のコントラストが WCAG AA（4.5:1）以上（red リンク・navy・text は満たす設計）
- PC・タブレット・スマホの全幅で1列表示になり、カード幅と余白が崩れない
- clamp が機能し、個別ページでは全文が出る
- キャラ大型表示がヘッダのみ（§5 の上限遵守）
